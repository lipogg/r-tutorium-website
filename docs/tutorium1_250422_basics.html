3.2 Variablen

Variablen werden verwendet, um Werte zu speichern, die wiederholt verwendet werden. Werte können mithilfe der Zuweisungsoperatoren -> und = einer Variable “zugewiesen” werden, das heißt, dass ein Name festgelegt wird, der für diesen Wert und für einen Speicherplatz steht. So kann mithilfe des Variablennamens auf ihren Inhalt zugegriffen werden, selbst dann, wenn sich der Wert im Programmverlauf verändert hat. Das Zuweisen eines Werts zu einer Variable wird auch Initialisieren genannt. In anderen Programmiersprachen muss eine Variable zuerst deklariert werden; dieser Schritt muss in R aber nicht separat ausgeführt werden. Bei der Benennung von Variablen sollten bestimmte Regeln und Konventionen eingehalten werden:

  • Variablen dürfen keine Sonderzeichen außer Punkte und Unterstriche enthalten.
  • Zwei Wörter oder ein Wort und eine Zahl können mit Unterstrich, Punkt, oder camelCase getrennt werden. Meine persönliche Empfehlung ist hier die Verwendung des Unterstrichs.
  • Variablen dürfen nicht mit einer Zahl oder einem Unterstrich anfangen.
  • R ist “case sensitive”, das heißt: die Variable baum ist nicht dasselbe wie die Variable Baum!
satz = "Eine Rose ist eine Pflanze"
satz_2 <- "Eine zweite Rose ist auch eine Pflanze"
noch_ein_satz = "Noch eine Rose"
Satz <- "Eine Rose ist keine Pflanze"

print(Satz)
## [1] "Eine Rose ist keine Pflanze"
print(satz)
## [1] "Eine Rose ist eine Pflanze"
satz = "Ein Apfel ist keine Pflanze"
print(satz)
## [1] "Ein Apfel ist keine Pflanze"

Verständnisfrage: Warum wird für die Variable satz das zweite Mal ein anderer Wert ausgegeben?

3.3 Datentypen

Die Werte der Variablen im Beispiel oben haben denselben Datentyp: character. Character ist kurz für “character string”; in anderen Programmiersprachen heißt dieser Datentyp daher oft “string”. Neben characters gibt es in R weitere Datentypen, z.B.

  • integer
  • numeric (“double”)
  • logical

In R gibt es verschiedene Funktionen, die erlauben, den Datentyp beispielsweise einer Variable auszugeben. Beachtet, dass im Beispiel unten die Funktion typeof(zahl_1) als Datentyp double ausgibt, während class(zahl_1) als Datentyp numeric ausgibt. Double ist, ähnlich wie bei character und string, der allgemeine Name für den Datentyp numeric. Der Default-Datentyp für Zahlen ist in R numeric. Das heißt immer, wenn ihr einer Variable einen numerischen Wert zuweist, wird dieser zunächst als numeric gespeichert. Die Funktion is.integer() gibt die Werte TRUE oder FALSE aus. Der Datentyp dieser Werte ist logical. Sie werden auch als Boolesche Wahrheitswerte bezeichnet.

zahl_1 = 1.5

zahl_2 = 1 

is.integer(zahl_2)
## [1] FALSE
is.numeric(zahl_1)
## [1] TRUE
typeof(zahl_1)
## [1] "double"
class(zahl_1)
## [1] "numeric"
x = 4.6
y = "4.6"

print(x)
## [1] 4.6
print(y)
## [1] "4.6"

Verständnisfrage: Enthalten die Variablen x und y den gleichen Wert?

3.4 Operatoren

In den bisherigen Beispielen haben wir bereits einen Operator kennengelernt, den sogenannten Zuweisungsoperator <- bzw. =. Daneben gibt es arithmetische Operatoren und logische Operatoren.

  • Arithmetische Operatoren sind aus der Schulmathematik bekannt. Es handelt sich dabei um Operatoren wie -, +, *, / sowie %% und %/%. Beachtet, dass der Divisionsoperator / nur für numerics definiert ist. Für die Division zweier integers wird ein spezieller Divisionsoperator, %/%, verwendet. %% ist der modulo Operator, damit kann bei einer Division der Rest berechnet werden. ^ oder ** wird zum Exponieren verwendet.

  • Logische Operatoren ermöglichen es, Werte zu vergleichen und die Richtigkeit einer logischen Aussage zu überpüfen. Die Vergleichsoperatoren <, >, <=, >= sind allgemein bekannt. Außerdem sind folgende Operatoren wichtig:

    Operator Bedeutung
    == exactly equal to
    != unequal to
    !x not x
    x ⎮ y x or y
    x & y x and y 
-5 < -10
## [1] FALSE
5 >= 5
## [1] TRUE
satz == Satz #hier sind werden die Variablen satz und Satz aus dem Beispiel oben verglichen
## [1] FALSE
x == y #wir können jetzt auch unsere beiden Variablen x und y vergleichen
## [1] TRUE
3 != 5 
## [1] TRUE
"Hallo" != "Hello"
## [1] TRUE
3 == 3 & "c" == "c"
## [1] TRUE
3 == 3 | "c" == "a"
## [1] TRUE
aussage_1 <- "c" == "c"
aussage_2 <- 3 == 5
aussage_1 == !aussage_2
## [1] TRUE

Verständnisfragen:

  • Warum wird der Ausdruck 3 == 3 & “c” == “c” zu TRUE evaluiert?
  • Warum wird auch der Ausdruck 3 == 3 | “c” == “a” zu TRUE evaluiert?
  • Was passiert im letzten Beispiel?

3.5 Datenstrukturen

Bisher haben wir Variablen genutzt, um einzelne Werte vom Typ character, integer, numeric oder logical darin zu speichern. Variablen können aber auch strukturierte Sammlungen von Werten enthalten. Solche Datenstrukturen können entweder nur Elemente vom selben Typ aufnehmen (wie z.B. sog. “atomare” Vektoren und Matrizen), oder Elemente unterschiedlichen Typs (wie z.B. Listen und Dataframes). Listen sind genauer gesagt eine spezielle Art Vektor mit der Eigenschaft, dass sie Elemente unterschiedlichen Typs enthalten können; genauso sind Dataframes spezielle Matrizen, die Elemente unterschiedlichen Typs beinhalten können. Im Unterschied zu Matrizen sind Vektoren eindimensional. Zwischen den beiden besteht aber ein Zusammenhang: Jede Zeile und jede Spalte in einer Matrix kann auch als Vektor aufgefasst werden.

Jedes Element in einem Vektor hat eine Position (auch “Index” genannt). Über diesen Index kann auf jedes Element einzeln zugegriffen werden. In R erfolgt der Zugriff mithilfe von eckigen Klammern (s.u.).

Datenstrukturen können mithilfe bestimmter Funktionen untersucht werden, beispielsweise gibt die length() Funktion die Anzahl der Elemente eines Vektors oder einer Liste aus.

#Vektoren erstellen
character() #erstellt einen leeren character-Vektor
## character(0)
character(4) #erstellt einen character-Vektor mit vier Elementen
## [1] "" "" "" ""
numeric(3) #erstellt einen numeric-Vektor mit drei Elementen
## [1] 0 0 0
leckere_fruechte = c("Apfel","Banane","Erdbeere")
coole_zahlen = c(45,90,7)

#Vektoren kombinieren
fruechte_und_zahlen = c(leckere_fruechte, coole_zahlen) #zwei Vektoren kombinieren
print(fruechte_und_zahlen)
## [1] "Apfel"    "Banane"   "Erdbeere" "45"       "90"       "7"
#Auf Elemente zugreifen 
erste_frucht = leckere_fruechte[1] #Zugriff auf das erste Element des Vektors leckere_fruechte
leckere_fruechte[1:3] #Zugriff auf Elemente 1-3
## [1] "Apfel"    "Banane"   "Erdbeere"
leckere_fruechte[c(1,3)] #Zugriff auf Elemente 1 und 3
## [1] "Apfel"    "Erdbeere"
#Matrizen erstellen
autos = matrix(c("Fiat", "Mercedes", "BMW", "Ford", "Volvo", "Audi"), ncol = 3)
print(autos)
##      [,1]       [,2]   [,3]   
## [1,] "Fiat"     "BMW"  "Volvo"
## [2,] "Mercedes" "Ford" "Audi"
#Dataframe erstellen: Variante 1 
Marke = c("Fiat", "Mercedes", "BMW", "Ford", "Volvo", "Audi")
Farbe = c("rot", "rot", "blau", "silber", "schwarz", "silber")
Jahr = c(1993, 2007, 1999, 2010, 2020, 2021)
df = data.frame(Marke, Farbe, Jahr)
print(df)
##      Marke   Farbe Jahr
## 1     Fiat     rot 1993
## 2 Mercedes     rot 2007
## 3      BMW    blau 1999
## 4     Ford  silber 2010
## 5    Volvo schwarz 2020
## 6     Audi  silber 2021
#Dataframe erstellen: Variante 2
df = data.frame(Marke = c("Fiat", "Mercedes", "BMW", "Ford", "Volvo", "Audi"),
                Farbe = c("rot", "rot", "blau", "silber", "schwarz", "silber"),
                Jahr = c(1993, 2007, 1999, 2010, 2020, 2021)
                 )

#Zugriff auf eine Spalte 
df$Farbe
## [1] "rot"     "rot"     "blau"    "silber"  "schwarz" "silber"
#Zugriff auf einzelne Elemente mithilfe des Indexes einer bestimmten Spalte
df$Farbe[3]
## [1] "blau"
#Zugriff auf einzelne Elemente abhängig von einem bestimmtem Wert in einer anderen Spalte
df$Marke[Farbe=="rot"]
## [1] "Fiat"     "Mercedes"
#Eigenschaften untersuchen
length(leckere_fruechte)
## [1] 3
nchar(erste_frucht) 
## [1] 5
length(df)
## [1] 3
ncol(df)

nrow(df)
## [1] 6
dim(df)

## [1] 2

3.6 Funktionen

Einige Funktionen haben wir bereits kennengelernt, beispielsweise print(), length(), typeof(). Beachtet, dass Funktionen meist nur auf bestimmten Datenstrukturen oder für konkrete Datentypen definiert sind, das heißt, dass eine Funktion nur korrekt ausgeführt wird, wenn ihr Argument beispielsweise ein character-Vektor oder ein Boolescher Wahrheitswert ist. Funktionen nehmen meist mindestens ein Argument an. Durch das Hinzufügen verschiedener Parameter kann das Verhalten von vielen Funktionen verändert werden.

Informationen zu einzelnen Funktionen und ihren Argumenten können direkt im R Studio über die Konsole mit dem Befehl ?funktionsname abgerufen werden.

?length
?print

3.7 Pakete

Funktionen wie print(), length() und typeof() sind bereits in R “vorinstalliert”, es handelt sich dabei um sogenannte R base Funktionen. Viele Anwendungen sind allerdings mithilfe von R base Funktionen umständlich oder sogar unmöglich. Deswegen gibt es viele verschiedene Pakete (engl. packages), die installiert und bei Bedarf geladen werden können. Im Rahmen dieses Tutoriums und des Seminars zur Digitalen Dramenanalyse werdet ihr verschiedene Pakete kennenlernen, zum Beispiel DramaAnalysis und stylo.

Für die meisten Pakete findet sich eine Dokumentation auf der Website https://cran.r-project.org/

3.9 Hilfe!!

Oft kommt es beim Coden zu komischen Fehlermeldungen, die mehr verwirren als helfen. Häufig steckt aber ein ganz simpler Flüchtigkeitsfehler dahinter. In diesem Fall gilt:

  1. Syntax und Rechtschreibung checken: Fehlt vielleicht nur eine Klammer? Ist die Variable wirklich richtig geschrieben? Sind wirklich alle notwendigen Pakete installiert und geladen?
  2. Fehlermeldung kopieren und googeln: Bestimmt hatte schonmal jemand anderes dasselbe Problem und bestenfalls findet sich eine Lösung auf https://stackoverflow.com/ oder in einem anderen Forum.

Manchmal macht das Skript aber auch einfach nicht das, was es soll, ohne, dass eine Fehlermeldung entsteht. In diesem Fall liegt wahrscheinlich ein logischer Fehler im Programmablauf vor. Für diesen Fall gibt es eine Strategie, die Rubber Ducking oder Quietscheentchen-Debugging genannt wird.

  1. Rubber Ducking oder Quietscheentschen-Debugging. Wenn nichts mehr hilft, hilft nur eins: Den Code einer Person, die nichts davon versteht - oder eben einem Quietscheentchen, Zeile für Zeile erklären. Dabei fallen oft logische Fehler auf, die das Problem verursachen.
  2. Hilfe holen. Falls ihr im Laufe des Tutoriums ein Problem habt, das ihr selbst nicht lösen könnt, könnt ihr einen Screenshot oder ein Codesnippet in das Diskussionsforum in unserer R-Tutoriumsgruppe hochstellen oder mir eine E-Mail an l.poggel@fu-berlin.de schreiben.
